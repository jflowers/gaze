## Why

`gaze report` panics when run under Go 1.25 against any module that transitively
depends on `github.com/go-json-experiment/json`. The panic originates in the SSA
builder inside `golang.org/x/tools@v0.42.0` (`go/ssa/subst.go:559`), which does
not handle generic variadic parameters with named types (e.g. `jsontext.Value`).
Go 1.25 exposes this latent bug â€” likely through stricter type representation in
`go/types`. The result is a hard crash during the quality (SSA) analysis phase of
`gaze report`, blocking CI integration for any project that requires Go 1.25.

Additionally, gaze's CI matrix only tests against Go 1.24, so this class of
regression goes undetected until users report it.

Tracking: GitHub issue #29.

## What Changes

- `go.mod`: bump `golang.org/x/tools` from `v0.42.0` to `v0.43.0` (released
  2026-03-12, contains the SSA fix) and update the `go` directive from `1.24.2`
  to `1.25.0` (required by `x/tools@v0.43.0`). Transitive upgrades: `x/mod`,
  `x/sync`, `x/sys`.
- `go.sum`: regenerated by `go mod tidy`.
- `.github/workflows/test.yml`: add a Go version matrix (`["1.24", "1.25"]`) to
  both the `unit-and-integration` and `e2e` jobs, replacing the hardcoded
  `go-version: "1.24"` with `${{ matrix.go-version }}`.
